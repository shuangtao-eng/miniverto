# Code of Conduct

Miniverto is an open-source learning tool. We want the project to be useful, calm, and welcoming for people who are learning, building, reporting issues, or reviewing ideas.

## Expected Behavior

- Be respectful and constructive.
- Assume good intent, especially with first-time contributors.
- Keep feedback specific, actionable, and focused on the work.
- Respect privacy. Do not post API keys, private materials, local files, or personal data in public issues.
- Use English as the main project language. Chinese is welcome when it helps clarify context.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Publishing someone else's private information without permission.
- Repeated off-topic comments that make project work harder.
- Security disclosures that expose private details publicly instead of following `SECURITY.md`.

## Reporting

If you see behavior that makes the project unsafe or unwelcoming, contact the maintainer through the repository owner profile. For security-sensitive reports, follow `SECURITY.md` and use GitHub private vulnerability reporting when available.

Reports will be reviewed with care. Maintainers may remove comments, close issues, block users, or take other reasonable action to protect the project community.
