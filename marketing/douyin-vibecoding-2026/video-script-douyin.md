Final rendered duration: about 136 seconds.

## Shot List And Voiceover

### 0-12s: Hook

Visual: `images/01-cover-miniverto-vibecoding.png`.

Voiceover:

```text
这是我的 VibeCoding 大赏投稿，作品叫 Miniverto。它不是简单生成一张学习计划表，而是一个会根据学习状态调整路线的 AI 学习系统。
```

### 12-43s: Data Collection

Visual: `images/02-learning-pain.png`, with an enlarged questionnaire detail panel.

Voiceover:

```text
第一步，它会通过以选择题为主的数据采集，了解你的学习目标，当前基础，可投入时间，学习偏好，已有资料，设备条件，阻碍因素和目标期限。问卷不是让你写一大段作文，而是把关键选择拆出来，比如你每周能学多久，是更喜欢文字，图片，视频，还是项目练习。这样做，是为了避免 AI 给出一份看起来很完整，但和你实际情况脱节的通用计划。
```

### 43-69s: Personalized Multimodal Plan

Visual: `images/03-learning-loop.png`, with an enlarged plan-structure panel.

Voiceover:

```text
第二步，Miniverto 会根据这些信息量身定做计划。计划不是一句建议，而是会拆成阶段目标，里程碑，每日或每周任务，材料建议，练习安排和验收标准。长期方向上，它会把文字，图片，视频等多种学习材料组织进同一条路线，让用户知道每个节点要学什么，为什么学，以及学完怎么检查。
```

### 69-92s: Adaptive Learning Nodes

Visual: `images/04-real-interface.png`, with an enlarged node-detail panel.

Voiceover:

```text
第三步，计划不是一次生成就结束。每个学习节点里，会有目标说明，推荐材料，练习任务，笔记区和小测入口。在学习过程中，任务完成情况，笔记内容，阶段测评，卡点反馈，时间变化和理解程度，都可以成为重新规划的依据。
```

### 92-114s: End-Of-Cycle Assessment

Visual: `images/05-assessment-loop.png`, with an enlarged assessment panel.

Voiceover:

```text
当一个学习周期完成后，Miniverto 会测试你的学习成果，判断是否达到最初目标。测评会看概念理解，应用题，迁移能力和自我解释。如果某些部分仍然薄弱，就建议加强对应模块；如果目标已经达成，就进入下一阶段学习。
```

### 114-136s: Local-First And Open Source

Visual: `images/06-open-source-invite.png`.

Voiceover:

```text
我把它做成桌面应用，并且坚持本地优先。学习目标，弱点，资料，笔记和 API Key 都很私人，所以项目数据默认留在本机。技术栈是 Tauri，React，Rust 和 SQLite。项目已经开源，GitHub 搜索 shuangtao-eng slash miniverto。
```
